# AI Trading Agent

Fullstack-система анализа крипторынка: AI-агент анализирует технические индикаторы, управляет виртуальными позициями и визуализирует решения в реальном времени. Включает собственный бэктест-движок для проверки торговых стратегий на исторических данных.

> **Ключевой результат проекта:** бэктест на 90 днях исторических данных показал, что стратегия на чистых технических индикаторах не даёт статистического преимущества (winRate ~37% при любых параметрах). Это задокументированный, осознанный вывод — проект демонстрирует не «прибыльного бота», а инженерный подход к проверке гипотез на данных.

## Стек

**Frontend**
- React 18 + TypeScript
- Vite
- styled-components
- recharts

**Backend**
- NestJS 11 (TypeScript)
- Prisma 7 + PostgreSQL
- Groq SDK (LLM — llama-3.3-70b)
- @nestjs/schedule (cron)
- class-validator

**Внешние интеграции**
- Binance API — исторические свечи для бэктеста
- CoinGecko / Binance — реальные цены
- RSS (Cointelegraph, Coindesk, Decrypt) — новостной поток через `rss-parser`

## Возможности

- **Анализ рынка** — 6 технических индикаторов (EMA, RSI, MACD, Bollinger Bands, MA5, тренд) с weighted confluence логикой
- **Управление позициями** — открытие/закрытие, ATR-based trailing stop, расчёт realized и unrealized P&L
- **Виртуальный кошелёк** — баланс, история транзакций, P&L по периодам
- **AI reasoning** — объяснение каждого решения на естественном языке через LLM
- **Бэктест-движок** — загрузка исторических данных, симуляция стратегии без look-ahead bias, метрики (winRate, Sharpe ratio, max drawdown)
- **Инфополе** — агрегатор крипто-новостей из нескольких RSS-источников с дедупом, LLM-переводом и AI-дайджестом ленты (см. отдельный раздел)
- **Реалтайм-дашборд** — графики цены с точками входа, динамика баланса, история решений, открытые позиции, панель новостей
- **Автоанализ** — cron каждые 5 минут по BTC/ETH/SOL

## Бэктест: методология и результат

Бэктест-движок проверяет стратегию на реальных исторических данных Binance с соблюдением корректной методологии:

- **No look-ahead bias** — сигнал формируется на свече `i`, вход исполняется по `open` свечи `i+1`
- **Позиционный размер** — `units = balance / entryPrice`, реалистичный P&L
- **Идентичная логика** — те же индикаторы и пороги, что в live-анализе

### Результаты (90 дней, BTCUSDT)

| Параметр | Диапазон | Результат |
|----------|----------|-----------|
| Таймфрейм | 5m, 1h | winRate ~37% на обоих |
| Порог confluence | 4.0 → 7.0 | winRate застрял на 36–38% |
| Направление | BUY / SELL раздельно | оба убыточны |
| ATR multiplier | 2.0 → 4.0 | убыток уменьшался, но не в плюс |

**Вывод:** winRate не превышает 40% ни при каких параметрах. Технические индикаторы на ценовых данных не дают edge на ликвидном рынке — к моменту формирования сигнала движение уже произошло. Бэктест-движок выполнил свою задачу: отсёк нежизнеспособную стратегию до вложения реальных средств.

Это и есть назначение бэктестинга — честно опровергать гипотезы.

## Инфополе: агрегатор новостей + AI-дайджест

После вывода о пределе технического анализа проект развивается в сторону **агрегации информационного поля** — сбора разнородных источников в единую картину. Первый реализованный слой — новостной.

### Архитектура агрегатора

```
Cointelegraph ─┐
Coindesk ──────┼─> NewsService ──> NewsTranslator ──> Digest ──> API ──> Dashboard
Decrypt ───────┘   (RSS parse,      (LLM-перевод      Service   /news    (NewsPanel)
                    дедуп, cron       + валидация)     (AI-свод)  /news/digest
                    каждый час)
```

- **Multi-source pull** — несколько RSS-фидов параллельно, нормализация в единый `NewsItem` (id, title, link, source, category, publishedAt).
- **Дедупликация** — новости из разных источников об одном событии схлопываются по стабильному id; лента не дублируется.
- **Часовой cron** — фон обновляет кэш ленты, эндпоинты отдают готовый результат без ожидания сети на запросе пользователя.

### AI-слой: перевод и дайджест

Над сырой лентой работают два LLM-этапа (Groq, llama-3.3-70b), и оба построены по принципу **«AI — не чёрный ящик»**: контролируется не только выход модели, но и форма выхода и частота отказов.

- **Перевод заголовков** (`NewsTranslator`) — батч-перевод с нумерацией 1:1, кэш «навсегда» (id заголовка неизменен) с FIFO-эвикшном. Выход валидируется: пустая строка или оставшиеся CJK-иероглифы = битый перевод → **fallback на оригинал**, результат не кэшируется, следующий refresh пробует заново. Счётчик `failCount` логирует частоту деградации — «молча вернули оригинал» перестаёт быть невидимым.
- **Дайджест ленты** (`DigestService`) — свод за час в 3–5 предложений, сгруппированный по темам (цена, регуляторка, технологии), с промптом на **обобщение тренда, а не пересказ заголовков**. Кэш по SHA-1 хешу набора id: лента не изменилась → модель не вызывается.

### Почему это инженерный, а не «прикрутил LLM» слой

| Проблема LLM-интеграции | Решение в проекте |
|-------------------------|-------------------|
| Недетерминированный выход | Валидация формы (`isValid`), а не только наличия |
| Тихая деградация до пользователя | Метрика `failCount` + логирование битых переводов |
| Стоимость повторных вызовов | Кэш перевода (FIFO) и дайджеста (хеш ленты) |
| Плохой результат «залипает» в кэше | Невалидный перевод не фиксируется → повторная попытка |
| Латентность на запросе юзера | Cron-прогрев кэша, эндпоинты читают готовое |

## Запуск

**Backend**
```bash
cd backend
npm install
npm run db:start        # PostgreSQL в Docker
npx prisma migrate dev
npm run start:dev        # http://localhost:3000
```

Переменные окружения (`.env`):
```
GROQ_API_KEY=...
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/trading_agent
```

**Frontend**
```bash
cd frontend
npm install
npm run dev              # http://localhost:5173
```

## Тесты

```bash
cd backend
npm run test
```

Бизнес-логика анализа покрыта unit-тестами (сценарии weighted confluence).

## Дальнейшее развитие

Технический анализ как источник сигнала исчерпан (см. вывод выше). Проект развернулся в сторону агрегации информационного поля — первый слой (новости + AI-дайджест) уже реализован (см. раздел «Инфополе»). Дальше — расширение источников: рынки прогнозов (Polymarket), on-chain активность, настроения соцсетей. Цель — аналитический инструмент, который собирает разнородные источники в единую картину для принятия решений человеком.
