Ты планируешь реализацию фичи `$1` для AI Trading Agent.

## Шаги

1. Прочитай `CLAUDE.md` — архитектура и текущее состояние проекта.
2. Прочитай `docs/$1.md` если существует — это ТЗ фичи.
3. Исследуй релевантные файлы в `src/` чтобы понять куда встраивается фича.
4. Создай `docs/plan/$1.md`:
    - **Компоненты** — какие модули/файлы создаются или меняются
    - **Контракты** — сигнатуры методов, формат данных
    - **Потоки данных** — как данные проходят через систему
    - **Риски** — что может пойти не так
    - **Trade-offs** — какие компромиссы приняты и почему
5. Создай `docs/tasklist/$1.md`:
    - Список задач с `- [ ]`
    - Каждая задача — маленький проверяемый шаг
    - Для каждой 1-2 критерия готовности

## Правила
- Задачи должны быть независимыми насколько возможно.
- Критерий готовности проверяемый: не "улучшить X", а "метод Y возвращает Z".
- Не пиши код — только планируй.
- Учитывай существующие паттерны проекта (NestJS, Prisma, confluence логика).
