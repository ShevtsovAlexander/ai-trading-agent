Ты ревьюишь реализацию фичи `$1` для AI Trading Agent.

## Шаги

1. Прочитай `docs/$1.md` (ТЗ), `docs/plan/$1.md` (план), `docs/tasklist/$1.md`.
2. Прочитай `CLAUDE.md` для понимания конвенций.
3. Проанализируй изменённый код по этой фиче.
4. Сформируй список замечаний по категориям:
    - **Blocking** — нужно исправить до использования (баги, нарушение логики)
    - **Important** — желательно поправить (производительность, читаемость)
    - **Minor** — косметика
5. Проверь:
    - соответствует ли код ТЗ и плану
    - не нарушены ли паттерны проекта
    - есть ли необработанные edge cases
    - корректна ли типизация

## Правила
- Не придирайся к стилю если он не нарушает конвенции.
- Фокус на: логика, безопасность, производительность, типы.
- Не правь код сам — только список замечаний.
- Если всё хорошо — так и скажи, не выдумывай проблемы.
